Use this checklist to cut the release without reopening scope.

## Scope Freeze

- [ ] Support claim remains:
  - supported: macOS + REAPER + VST2
  - experimental: Windows, Linux, VST3, AU v2, 32-bit
- [ ] No release prose implies broader support than the validation matrix
- [ ] Known issues still match the intended release wording

## Release Candidate State

- [ ] worktree clean
- [ ] `effigy qa` passes
- [ ] `effigy doctor` passes
- [ ] latest target commit has green CI
- [ ] `CHANGELOG.md` and release docs match the target commit

## Artifact Build

- [ ] build the release candidate
- [ ] collect macOS artifact
- [ ] decide whether Windows and Linux artifacts ship in this alpha drop or are
      omitted from the published artifact list
- [ ] verify helper-binary placement for every shipped artifact
- [ ] generate checksum manifest

## Release Notes

- [ ] release title uses `v0.1-alpha`
- [ ] highlights are copied from the validated release doc
- [ ] support scope is explicit
- [ ] known issues are linked
- [ ] install instructions are present
- [ ] experimental lanes are called experimental

## Final Verification

- [ ] install from the packaged artifact on the primary alpha platform
- [ ] launch REAPER
- [ ] verify one representative bridged plugin appears
- [ ] verify plugin UI opens
- [ ] verify short transport playback
- [ ] capture one post-package proof note

## Publish

- [ ] create tag `v0.1-alpha`
- [ ] publish GitHub release
- [ ] attach artifacts
- [ ] attach checksums
- [ ] verify release page renders correctly

## Next Task

Use this checklist against one release-candidate commit, then publish only if
the packaged artifact still matches the validated primary support lane.
