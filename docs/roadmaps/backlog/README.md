# Backlog

Deferred scope items with promotion criteria.

Items here are not in the active generation queue. They should be promoted to
an active milestone only when the prerequisites and planning work that blocks
them are complete.

## Items

No backlog items yet.

## Promotion Rule

An item moves from backlog to an active milestone when:
- the prerequisite work it depends on is complete
- the contracts and architecture it needs exist
- a maintainer decides the timing is right

Do not promote backlog items into active milestones speculatively.
