# G01.009 — 32-Bit Bridge Binaries

Date: 2026-04-11
Milestone: g01.009
Status: complete (infrastructure ready, 32-bit build needs multilib toolchain)

## What was done

### IPC struct safety (step 2)

- Added `#pragma pack(push, 1)` / `#pragma pack(pop)` around all IPC
  payload structs to ensure identical layout across 32/64-bit
- Added `static_assert` size checks for every IPC struct:
  IpcHeader (8), IpcActivatePayload (12), IpcProcessPayload (4),
  IpcSetParamPayload (8), IpcMidiEventPayload (8), IpcPluginInfo (28),
  IpcParamInfoResponse (88), IpcEditorRect (8)
- No pointer-sized fields in any IPC struct (all use fixed-width int32/uint32/float/double)

### Architecture detection (step 3)

- Added PE header inspection for Windows DLLs: reads MZ+PE signature,
  checks IMAGE_FILE_MACHINE_I386 (0x014C), AMD64 (0x8664), ARM64 (0xAA64)
- Added ELF header inspection for Linux .so: checks EI_CLASS,
  e_machine for EM_386 (3), EM_X86_64 (62), EM_AARCH64 (183)
- Existing Mach-O detection (thin + fat binaries) unchanged
- All detectors return "native", "x86", "x86_64", "arm64", or nullptr
- Scanner uses platform-appropriate detector: Mach-O on macOS, PE on
  Windows, ELF on Linux

### Bridge routing (steps 4–5)

- Factory now tracks `needs_32bit_bridge` flag per plugin
- Three bridge binary paths: `s_bridge_path` (native), `s_bridge_x86_64_path`
  (Rosetta), `s_bridge_32_path` (32-bit)
- `create_plugin` selects the correct bridge: 32-bit → `keepsake-bridge-32`,
  x86_64 cross-arch → `keepsake-bridge-x86_64`, native → `keepsake-bridge`
- Windows: `keepsake-bridge-32.exe`, Linux: `keepsake-bridge-32`

### Build infrastructure (step 1)

- CMake toolchain file: `cmake/toolchain-linux-x86.cmake` for 32-bit Linux
  builds (requires `gcc-multilib`)
- CMake preset: `linux-x86` for 32-bit Linux configuration
- Windows 32-bit: standard MSVC cross-compilation (no toolchain file needed,
  use `-A Win32` with CMake)

## Known limits

- 32-bit binaries not built or tested (requires Linux multilib or Windows
  cross-compilation environment)
- macOS has no 32-bit support (10.15+ dropped it) — documented as platform
  limitation
- Actual 32-bit plugin testing deferred to CI (g01.007)
